| Domain | Exact candidate paths and disposition |
|---|---|
| Planning | Retire lane-status authorization in `scripts/ai/lib/round_aggregate.py`; retire Markdown-existence landing, legacy manifest synthesis and `_LIB_OK` fail-open in `scripts/ai/aq-collab-round`; demote dispatch/agent/aggregate Markdown to evidence projections |
| Delegation/session | Consolidate direct registry writers in all `delegate-to-*`; retire `aq-agent-reap` as second reconciler after durable repair; retire `scripts/ai/lib/pending-update`; separate Intent Lock from delegation; remove direct RESUME/PULSE writes in `loop_state.py`, `aq-loop-queue`, and `aq-local-training-loop` |
| Workflow | Retire `scripts/ai/sync-workflow-sessions.py`; remove same-file writes from `workflow_executor.py`; consolidate or distinctly type graph-run JSON; either wire `workflow_checkpointer.py` or retire dormant recovery claims |
| QA/effectiveness | Replace both shared-latest writers; retire `_synthesize_scorecard_from_report()` in dashboard `aistack.py`; migrate remediation/readers to verified immutable invocation artifacts |
| Routing | Retire streaming direct-llama fallback in `local_agent_runtime.py`; retire direct llama branches in `route_handler.py`; route eval/context/model helpers through switchboard; remove payload/fallback copies after parity |
| Learning/eval | Consolidate service/fallback loop JSONL and dual spools; stop dashboard mtime adjudication; ingest or retire `model_optimization.py` parallel corpora; define Postgres eval relationship before authority claim |
| Memory | Gate direct Qdrant writers in `seed-rag-knowledge.py`, AIDB server, `collective_memory.py` and qdrant-populator behind broker/collection ownership; retire ad-hoc Markdown seeding after lineage ingestion |
| Config | Retire/rename dashboard process-local `/api/config`; remove legacy profile paths, compatibility aliases, duplicated endpoints and fallback maps only after usage telemetry |
| Operator | Persist complete approvals in SQLite and retire approval dictionaries; rebuild task caches from durable state; consolidate/remove generic shell action catalog; retain UI/badges/WebSocket/audit log as projections/evidence |

Retirement here is a decision candidate, not deletion authority. Exact archive actions require reference
scans, usage/divergence evidence, owner, rollback and the plan's clean-cycle deadline.

`VERDICT: SPLIT_BRAIN CONFIRMED — individual modules have useful local authorities, but every requested
cross-system lifecycle has multiple writers, incompatible projections, or an unowned recovery boundary.`
