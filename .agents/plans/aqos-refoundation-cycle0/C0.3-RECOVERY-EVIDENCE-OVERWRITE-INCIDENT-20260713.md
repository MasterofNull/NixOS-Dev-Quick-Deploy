## Root-control gap and required fix

The fixed evidence filename is a multi-writer race surface. Prompt-level role restrictions did not
prevent an external lane from producing implementation evidence after owner activation. Before any
new attempt:

- allocate a unique run-scoped evidence path and bind it before execution;
- make the evidence directory single-writer for the authorized principal;
- write into an isolated cell and publish once through a compare-and-swap/hash-bound broker;
- reject any existing target, unexpected inode change, or writer identity;
- disable or fence autonomous drop-zone/IDE implementation triggers for the slice;
- make the reviewer consume an immutable content-addressed artifact rather than a mutable filename.

The original authorized evidence summaries are not sufficient for acceptance because the bytes were
destroyed. They must not be reconstructed or promoted.
