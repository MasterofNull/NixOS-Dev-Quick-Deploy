## 2. Invariant & Requirement Checklist

### 1. Seven-File R0 Boundary & No Live Overlap
- The R0 scope is strictly confined to the exact seven-file inventory:
  1. `.agent/PROJECT-LOCAL-DELEGATION-RELIABILITY-PRD.md`
  2. `.agent/PROJECT-LOCAL-DELEGATION-RELIABILITY-PLAN.md`
  3. `config/schemas/local-delegation-runtime-policy.schema.json`
  4. `config/local-delegation-runtime-policy.json`
  5. `scripts/ai/lib/local_delegation_reliability.py`
  6. `scripts/testing/fixtures/local-delegation-reliability-golden.json`
  7. `scripts/testing/test-local-delegation-reliability.py`
- Bidirectional adoption guard (`test_14_bidirectional_adoption_guard`) validates that there is no import overlap or leakage into production files (such as `dispatch.py` or staged `L2B-A` candidates).

### 2. D1–D11 Defect Static Characterization
- `execute_golden` and `test_02` verify that all 11 defects (`D1` through `D11`) are characterized by frozen regex rules and predicates against the live codebase.
- Stable digests are generated by the pure module and locked in `local-delegation-reliability-golden.json` for:
  - `vectors` (identity, budget, phase, fairness)
  - `source_manifest`
  - `fixed_contract`

### 3. Idempotent TERM/KILL & Terminal Publication
- The `Cancellation` class enforces that `sigterm_sent` and `sigkill_sent` transitions are idempotent and sequence-locked (`test_09`).
- `terminal_publications` is strictly capped at `1` to prevent multiple conflicting terminal states from being broadcast or recorded in the delegation registry.

### 4. Adversarial CAS, Fencing, PID Reuse & Tampering
- **Registry CAS**: Thread pool concurrency stress tests (`test_11`) verify that under a 1,000-mutation concurrent workload, the Registry's CAS logic correctly rejects stale generations and prevents lost updates.
- **Process Fencing**: Descriptor-held kernel locks (via `fcntl.flock`) and inode checks prevent concurrent writer overlap. Stale writers attempting updates are blocked.
- **PID Reuse Protection**: Validation requires the combination of PID and process start time (`ProcessIdentity` matching `/proc`), preventing signaling to recycled PIDs.
- **Telemetry Tamper Rejection**: The `ProgressValidator` (`test_08`) rejects forged producers, duplicate transaction sequences, clock drift anomalies, and replayed records.

### 5. Closed JSON Schema (Draft-2020-12) & Undeclared Env
- The policy schema (`local-delegation-runtime-policy.schema.json`) enforces `additionalProperties: false` on every section and validates strictly against Draft-2020-12.
- Environment override validation (`resolve_environment_override`) rejects any undeclared environment variable alias to block out-of-band config injection (`test_13`).

### 6. Subprocess Hermeticity & State Protection
- The isolated test runner (`test_15`) spawns the pure module under strict `resource` limits (CPU = 2s, RSS = 256MB, FSIZE = 64KB, NPROC = 1) in a temporary directory with a sanitized environment.
- Setup and teardown functions (`setUpClass` and `tearDownClass`) ensure that the test suite does not modify the frozen codebase, nor does it mutate the state directories (`.agents/delegation` or `.agent/collaboration`).

### 7. Low-Cardinality Monitoring Vocabulary
- The metric set defined in `local-delegation-runtime-policy.json` contains exactly 19 variables.
- All metric names are low-cardinality (no dynamic IDs or run-specific content like `run_id` or `prompt_tokens`), conforming to the SRE policy requirements.
