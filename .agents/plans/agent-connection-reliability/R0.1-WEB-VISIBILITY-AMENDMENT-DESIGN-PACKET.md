## 3. Architecture and Single-Adapter Boundary

The one-way source path is:

```text
registry snapshot
  -> task_registry R0.1 compatibility scanner (sole JSONL parser; single bounded scan)
  -> aq.registry-compatibility-facts.v1 (sanitized only)
  -> aq-tui-dashboard --registry-compatibility-json (sole sanitized web adapter)
  -> dashboard backend adapter/cache
  -> GET /api/agent-ops/registry-compatibility
  -> Registry Compatibility card
```

Rules:

1. `scripts/ai/lib/task_registry.py` remains the sole authority allowed to open and parse registry JSONL. Its R0.1 compatibility scanner performs the one bounded scan and returns only sanitized facts to callers; the TUI does not implement framing, JSONL parsing, duplicate-key handling, identity classification, or record-size policy.
2. `scripts/ai/aq-tui-dashboard` is the sole sanitized web adapter. Its new machine mode calls the same `task_registry` compatibility-facts function used by the full Agent Ops projection and emits only the closed compatibility object.
3. The machine mode must not emit the full Agent Ops projection, `work` records, record objects, or target outcomes. Its stdout is a single closed JSON document; diagnostics contain fixed codes only.
4. The dashboard backend invokes that mode by absolute script path and argument vector. It must not import `task_registry`, open the registry, duplicate JSONL parsing, invoke the lookup CLI for a task, or pass a task ID.
5. The backend does not call the full `aq-tui-dashboard --json` mode because that would unnecessarily cross the process boundary with work identifiers.
6. No coordinator route or coordinator auth-list edit is needed. This is a dashboard-local, read-only projection of an existing host adapter.
7. No lifecycle transition, registry repair, migration, lock, temp registry, second store, or mutation endpoint is introduced.

### External web-coverage authority — no self-attestation

Neither `aq.registry-compatibility-facts.v1`, the TUI adapter, nor the web response may claim that the web dashboard is installed, reachable, visible, or healthy. A payload served by the route cannot be authoritative evidence that the route/card deployment is correct; using it that way would be circular self-attestation.

The existing `dispatch_contract.coverage_health` remains dispatch-only and unchanged. This amendment supersedes R0.1's proposed registry `coverage_health.web_dashboard` self-report: no registry-specific web-coverage field is added to the facts or response. The authoritative web-coverage evidence is external and immutable for the reviewed deployment subject: successful live Phase-0 `0.10.40`, exact deployed revision/hash, service identity/interpreter proof, visible-card assertion, and independent acceptance. R0.1 remains incomplete unless that external evidence exists even when the compatibility payload itself is healthy.

## 4. Closed Web API v1

Canonical route:

```text
GET /api/agent-ops/registry-compatibility
```

The existing dual router mount also provides `/api/aistack/agent-ops/registry-compatibility` as a compatibility alias. Both return the same body. The frontend uses the canonical route.

Schema identifier:

```text
aq.agent-ops.registry-compatibility-web.v1
```

Closed response:

```json
{
  "schema_version": "aq.agent-ops.registry-compatibility-web.v1",
  "generated_at": "2026-07-16T00:00:00Z",
  "state": "healthy|degraded|blocked|unavailable",
  "freshness": "fresh|stale|unavailable",
  "source_age_seconds": 0,
  "registry_compatibility": {
    "contract_version": "aq.registry-compatibility-facts.v1",
    "lookup_health": "healthy|degraded|blocked|unavailable",
    "strict_mutation_health": "healthy|blocked|unavailable",
    "legacy_oversized_rows": 0,
    "ambiguous_or_corrupt_rows": 0,
    "max_legacy_bound_bytes": 65536,
    "reason_codes": []
  },
  "reason_codes": []
}
```

`generated_at`, `source_age_seconds`, `legacy_oversized_rows`, and `ambiguous_or_corrupt_rows` are nullable. `generated_at` is null when no source document was accepted or an adapter-integrity envelope is returned; when present it is the backend's timezone-aware RFC 3339 wall time captured at successful acceptance, never an upstream field or cache authority. `max_legacy_bound_bytes` is a contract constant, not observed row data. Numeric counts are capped at the upstream contract maximum. Reason codes are closed, unique, low-cardinality enums with at most 32 entries and 64 ASCII characters each.

The route returns HTTP 200 for every typed health state so no-data and blocked health remain visible. Transport failures before the route are ordinary HTTP failures. It sets `Cache-Control: no-store`; the bounded server-side cache below is the sole response cache.

An incompatible response change requires a new schema and route version. Unknown fields are rejected by tests; v1 does not become open-ended through additive fields.

## 5. Health Semantics

### Healthy

- Source is fresh and contract-valid.
- `lookup_health=healthy` and `strict_mutation_health=healthy`.
- Both counts are exactly zero.

### Degraded — migration required or bounded stale evidence

- Fresh compatibility facts prove unrelated lookup remains safe, while bounded legacy rows make strict mutation blocked; or
- the last valid response is older than the fresh-cache TTL but no older than the stale ceiling.

The card visibly says **Migration required** for the first case and **Stale source** for the second. A stale previously blocked response remains blocked; staleness never improves severity.

### Blocked

- a fresh, schema-valid source reports identity ambiguity, corruption, compatibility-bound violation, or blocked lookup.

Blocked is not rendered as unavailable or zero. The card says **Lookup blocked** and may display only mapped fixed reason labels.

Adapter overflow, invalid JSON/schema, forbidden keys, or privacy canaries are **integrity/privacy failures**, not a registry-health observation. They return a fresh closed `blocked` adapter envelope with `generated_at`, `source_age_seconds`, and all source counts null; source health enums are `unavailable`, the fixed reason is `adapter_integrity_invalid`, the in-process accepted cache is invalidated, and an older cached body may never be served.

### Unavailable / no data

- No valid source sample has ever been accepted; or
- the TUI adapter is missing, cannot spawn, times out, exits nonzero without producing an integrity-invalid body, or the last valid cache is older than the stale ceiling; or
- upstream compatibility health is explicitly unavailable.

Counts are `null`, never fabricated as zero. The card says **No compatibility data**. Unavailable never becomes healthy merely because the HTTP route responded.

## 6. Polling, Cache, and Resource Bounds

### Backend adapter

- Invoke with `sys.executable`, absolute script path, `--registry-compatibility-json`, fixed repository working directory, no shell, closed stdin, and an environment allowlist that excludes API keys and secret-file variables.
- Hard timeout: **5 seconds**.
- Maximum stdout: **8 KiB**. Maximum stderr consumed: **2 KiB**. Start independent bounded reader tasks immediately for both pipes and drain them concurrently in fixed-size chunks; never call `communicate()` or drain one pipe before the other. If either cumulative limit is exceeded, stop retaining bytes, cancel both readers, terminate the child/process group, wait at most 500 ms, then kill and unconditionally await/reap it. After child cleanup, unconditionally await both reader tasks with bounded cancellation handling so neither task, pipe, exception, or buffer survives the request. Apply the same cancel-readers/terminate/wait/kill/reap/await-readers sequence on timeout, cancellation, decode failure, and route shutdown. No overflow bytes or stderr content enter logs or responses.
- Accept exactly one JSON document and validate both `aq.registry-compatibility-facts.v1` and the web v1 envelope.
- Fresh cache TTL: **15 seconds**.
- Stale-if-error ceiling: **60 seconds** from backend acceptance.
- Cache authority is an internal `accepted_at_monotonic` captured only after full stream bounds, privacy checks, and both schemas pass. Fresh/stale/expired decisions and `source_age_seconds` use elapsed monotonic time only; wall time, file mtimes, response time, and system wall-clock rollback are never cache authority.
- At that same successful acceptance boundary, the backend generates `generated_at` from its own UTC wall clock in normalized RFC 3339 form. R0.1 facts contain no timestamp and the TUI machine document adds none. Wall-clock rollback or jumps may affect only the display value of a later accepted sample; they never alter an existing cache age, stale ceiling, or health classification.
- Stale cache may be served only after an operational failure: missing executable, spawn/permission failure, bounded timeout, or nonzero exit with otherwise non-overflowing/non-sensitive output. Integrity, privacy, schema, JSON, forbidden-field, or stream-overflow failures invalidate/bypass the cache and return the closed blocked adapter envelope.
- At most one adapter subprocess may run per dashboard worker. Concurrent requests await the same in-flight task through one async lock/future.
- No retry inside one request and no cross-worker file cache, daemon, lifecycle store, or background timer.

### Production service context

The live dashboard authority is `command-center-dashboard-api.service`, running as the configured `cfg.primaryUser` and launched by the exact Nix-store `${dashboardPython}/bin/uvicorn` interpreter from `nix/modules/services/command-center-dashboard.nix`. Implementation must use `sys.executable` from that service process, not `/usr/bin/env python`, a developer shell, or an assumed PATH interpreter.

Before activation, external deployment evidence must capture `systemctl show` values for `User`, `Group`, `MainPID`, and `ExecStart`, resolve `/proc/<MainPID>/exe`, and prove through the live GET route that this exact service interpreter and service user can execute the sanitized adapter and import its R0.1 dependencies under the deployed AppArmor/systemd sandbox. The API must not expose the username, interpreter path, or import diagnostics. If the proof requires adding a Python dependency, PATH entry, `ReadOnlyPaths`, AppArmor rule, Nix package, wrapper, or service setting, stop: those files are outside this inventory and require a reviewed inventory amendment before implementation continues.

### Frontend

- Load once when the Intelligence lens is activated and thereafter only through the existing active-lens 60-second refresh path.
- Add no dedicated interval, WebSocket, or background poll.
- Reuse the existing eight-second `apiFetch` bound and add a single-flight promise so manual and lens refreshes cannot overlap.
- On fetch failure, render unavailable without retaining old values as current.

## 7. Security and Privacy Contract

Forbidden anywhere in stdout, API bodies, frontend DOM, logs, exceptions, snapshots, metrics, or fixtures:

- task IDs or work IDs;
- registry record content or raw JSON lines;
- descriptions, prompts, output, provider payloads, or parser excerpts;
- registry paths, line numbers, per-row byte sizes, inode/device values, PIDs, or usernames;
- secret values, environment contents, authorization headers, or key-file paths;
- arbitrary exception prose.

The backend allowlists keys and enums rather than recursively forwarding source JSON. Frontend rendering maps enums/reason codes to fixed labels and renders numbers only; it never injects source strings through `innerHTML`. Tests use synthetic canaries and assert absence at the TUI stdout, backend response, captured logs, and rendered DOM boundaries.

The endpoint is read-only, accepts no query/body/task identifier, and performs no network call beyond the browser's loopback dashboard request. Rate limiting and operator audit middleware remain inherited from the existing dashboard application.

Every case proves concurrent draining, overflow classification, terminate/wait/kill/reap, no zombie, no retained overflow bytes, and no content leak.
3. Timeout, caller cancellation, application shutdown, reader exception, decoder failure, and overflow on either pipe all exercise the same idempotent cleanup state machine. A child process group with a synthetic descendant proves the bounded group cleanup does not leave an orphan.
