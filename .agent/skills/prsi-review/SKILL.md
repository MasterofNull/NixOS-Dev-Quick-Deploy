# Skill: PRSI Autonomous Review

- **Purpose**: Autonomously review and approve or reject actions generated by the Pessimistic Recursive Self-Improvement (PRSI) queue.
- **Variables**: 
  - `queue_file`: Path to the PRSI action queue (e.g., `/var/lib/nixos-ai-stack/prsi/action-queue.json` or local snapshot).
  - `action_id`: (Optional) Specific action ID to review. If omitted, review all pending actions.
- **Instructions**:
  1. Retrieve pending PRSI actions using the `get_prsi_pending` tool or by reading the `queue_file`.
  2. For each pending action, analyze its `type`, `risk_level`, and proposed `summary`.
  3. Validate the action against system constraints (e.g., does it violate temporal supply-chain guards? Does it introduce breaking Nix module changes?).
  4. Use `prsi_orchestrate` with command `"approve"` (providing your agent name in `by` and a technical `note`) for safe actions.
  5. If an action is unsafe, leave it unapproved and log a rejection reason in your report.
  6. Finally, execute the approved actions using `prsi_orchestrate` with command `"execute"`.
- **Workflow**:
  1. `get_prsi_pending`
  2. Loop over actions -> `prsi_orchestrate {command: "approve"}`
  3. `prsi_orchestrate {command: "execute", dry_run: false}`
- **Report**: A summary of actions approved, actions rejected, and the execution status of the batch.
