# Integration Contract: Training Pipeline Health API (173-A -- 173-E)
**Status:** PROPOSED

## Endpoint: GET /api/aistack/training/health

### Description
This endpoint provides real-time health and throughput metrics for the continuous learning and fine-tuning pipeline. It is shared between the Data Engineering pipeline (173-A) and the Dashboard UI (173-E).

### Response Schema (JSON)
| Field | Type | Description |
|-------|------|-------------|
| dataset_size | integer | Total number of valid examples in the fine-tuning JSONL dataset. |
| ingest_rate_24h | float | Average ingestion rate (samples/hour) over the last 24 hours. |
| rejection_rate_24h | float | Percentage of telemetry events rejected by the quality/deduplication filters. |
| ragas_sample_count | integer | Number of high-quality samples available for RAGAS evaluation. |
| ragas_status | string | Status of the evaluation: `OK` (>=20 samples), `PRELIMINARY` (<20 samples), `INSUFFICIENT` (0 samples). |
| last_ingest_ts | string (ISO) | ISO 8601 timestamp of the last telemetry event successfully processed. |
| tool_result_samples | integer | Total count of tool-call patterns extracted and indexed. |

### Example Response
```json
{
  "dataset_size": 1250,
  "ingest_rate_24h": 42.5,
  "rejection_rate_24h": 12.3,
  "ragas_sample_count": 25,
  "ragas_status": "OK",
  "last_ingest_ts": "2026-06-17T14:30:00Z",
  "tool_result_samples": 840
}
```

### Implementation Notes (binding on both owners)
1. `tool_result_samples` — count of lines in `dataset.jsonl` where `source == "tool_result"`.
   Backend route reads the file directly (`grep | wc -l` equivalent). No coordinator API call.
2. `last_ingest_ts` — max timestamp of events in dataset.jsonl, NOT filesystem mtime.
   mtime changes on any write; ingest_ts must reflect actual training data provenance.
3. `ragas_sample_count` + `ragas_status` — sourced from coordinator health endpoint or
   a shared state file written by continuous_learning.py (173-B's responsibility to expose it).
   173-E must not hardcode these values.
4. `rejection_rate_24h` — count of events that passed `_is_useful_hybrid_event()` but failed
   quality scoring, divided by total events seen in last 24h. training_ingest.py dry-run mode
   can emit this; alternatively estimate from dataset growth rate vs telemetry volume.
